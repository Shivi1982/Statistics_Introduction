Set Theory:¶

In order to understand Conditional Probability we first need to understand some basics of Set Theory

Set:¶

A Collection of numbers without repeating a single number more than a single time.
Set is represented as A = {1,2,3,4,5}

Here A is a set.

Union of Set¶

So Union is an Operation and it need operands and operands are sets.
For Example, A = {1,2,3,4} B = {5,6,7,8}

A U B = {1,2,3,4,5,6,7,8}

Intersection Of Set¶

Intersection contains only common elements between two sets
For example: A = {1,2,3,4} B = {2,3}
A ꓵ B = {2,3}

Conditional Probability¶

It is a type of Probability that is calculated when another event which is given and has already occured
For Example, Given that it rained yesterday, what is the probability of having rain today.

And its Relation is:
Conditional Probability = P of (A ꓵ B) / P (B)
[Probability of B given A]

Conditional Prob of Independent Events¶

  • Independent Events: When there are two events and Occurence of one Event doesn't affect the probability of another event.
  • The outcome of tossing the first coin cannot influence the outcome of tossing the second coin
  • P(A|B) = P(A) conditional probability is the probability of an event A occurring given that event B’s already occurred

Bayes' rule (or theorem) ¶

METHODS THAT MODEL THE JOINT DISTRIBUTION OF Y AND PREDICTORS X ARE REFERRED TO AS GENERATIVE MODELS. nAIVE BAYES, LDA AND QDA [ quadratic discrimant analysis ]

Bayes’ theorem (alternatively Bayes’ law or Bayes’ rule) describes the probability of an event, based on prior knowledge of conditions that might be related to the event.
For example, if a disease is related to age, then, using Bayes’ theorem, a person's age can be used to more accurately assess the probability that they have the disease, compared to the assessment of the probability of disease made without knowledge of the person’s age.
Bayes’ theorem is stated mathematically as the following equation:
$$ P(A\mid B)={\frac {P(B\mid A).P(A)}{P(B)}} $$
where $A$ and $B$ are events and $P(B)\neq0$.

  • $P(A\mid B)$ is a conditional probability: the likelihood of event $A$ occurring given that $B$ is true and has occured.
  • $P(B\mid A)$ is also a conditional probability: the likelihood of event $B$ occurring given that $A$ is true and has occured.
  • $P(A)$ and $P(B)$ are the probabilities of observing $A$ and $B$ independently of each other; this is known as the marginal probability.

Sensitivity and Specificity¶

__The ideal test should be able to deliver results with 100% sensitivity and 100% specificity. However, in a practical application, it is difficult to achieve this.__
specificity is very important in medical testing and chemical testing. In medical testing, confirming that a person does not have the disease is more important than detecting whether a person has it

Suppose that a test for using a particular drug is 97% sensitive __RECALL RATE__ and 95% specific.
Sensitive = accurately predicting true cases
Specific = accurately predicting False cases
That is, the test will produce 97% true positive results for drug users and 95% true negative results for non-drug users.

Suppose that 0.5% of the general population are users of the drug. What is the probability that a randomly selected individual with a positive test is a drug user?

We will write a custom function which accepts the test capabilities and the prior knowledge of drug user percentage as input and produces the output probability of a test-taker being an user based on a positive result.

The Bayes' rule based calculation is given by the following equation,

$$ P(\text{User}|+)=\frac{P(+|\text{User}).P(\text{User})}{P(+)}\\ = \frac{P(+|\text{User}).P(\text{User})}{P(+|\text{User}).P(\text{User})+P(+|\text{Non-user}).P(\text{Non-user})} $$

Here,

$P(\text{User}) = \text{Prevelance rate}$
π
$P(\text{Non-user}) = 1-\text{Prevelance rate}$
$P(+|\text{User})=\text{Sensitivity}$
$P(-|\text{Non-user})=\text{Specificity}$
$P(+|\text{Non-user})=1-\text{Specificity}$

def drug_user(prob_th=0.5,sensitivity=0.85,specificity=0.90,prevelance=0.02,verbose=True):
    """
    """
    p_user = prevelance
    p_non_user = 1-prevelance
    p_pos_user = sensitivity
    p_neg_user = specificity
    p_pos_non_user = 1-specificity
    
    num = p_pos_user*p_user
    den = p_pos_user*p_user+p_pos_non_user*p_non_user
    
    prob = num/den
    
    if verbose:
        if prob > prob_th:
            print("The test-taker could be an user")
        else:
            print("The test-taker may not be an user")

    return prob

# Calculate Probability

Even with a test that is 85% correct for catching positive cases, and 90% correct for rejecting negative cases, the true probability of being a drug-user with a positive result is only 14.8%!

If you look at the computations, this is because of extremely low prevelance rate. The number of false positives outweighs the number of true positives.

For example, if 1000 individuals are tested, there are expected to be 995 non-users and 5 users. From the 995 non-users, 0.05 × 995 ≃ 50 false positives are expected. From the 5 users, 0.95 × 5 ≈ 5 true positives are expected. Out of 55 positive results, only 5 are genuine!

Conclusion¶

This conclusion can be intuitively derived from the fact that the main issue with having low probability is the low prevelance rate. Therefore, catching non-users correctly (i.e. improving specificity) is the area where we should focus on because they are much larger in numbers that the user.

Negative examples >> Positive examples in this problem. Therefore, True Negative performance of the test should be excellent.

Linear Regression¶

  1. In regression, the intercept only model has no independent variables. Thus, it predicts that the best estimate of the dependent variable for each subject is the overall mean. Often, the use of the intercept only model is as a comparison to other models that do include independent variables
  2. The intercept (often labeled the constant) is the expected mean value of Y when all X=0
    • If X never equals 0, then the intercept has no intrinsic meaning. In scientific research, the purpose of a regression model is to understand the relationship between predictors and the response
    • The formula for regression can be xpressed in 2 ways [is the same but represented in 2 ways majorly]:
      Y = a + bX + error, where the slope of the line is b, while a is the intercept.
  3. The slope of the line represents the change of the data plotted on the y-axis to the change of the data plotted on the x-axis
  4. __Coefficient = A number used to multiply a variable.Example: 6z means 6 times z, and "z" is a variable, so 6 is a coefficient__

Measures of Performance¶

  1. Mean Squared Error [MSE]: first we take the diff in actual vs predicted, square it and then take the average
  2. R-Sq.[Coefficient Of Determination] : Explains how much total variance in the data is explained by the model
  3. Adjusted R Sqr: When we add more independent variables into the model, R sqr increases irrespective of whether the additional variables improve the model.
    • __Hence Adjusted R Square penalises the model if the new variable doesnt fit the model__
  4. Mallow's Cp: used in selecting the best regression model, its value should be small and close to the number of independent variables

Multi-Collinearity¶

A phenomenon in which 2 or more predictor variables are in mutiple regression model are highly correlated.
This can be detected using:
1- Correlation Matrix
2- VIF {Variation Inflation Factor} - if none of the independt var are correlated then VIF will be 1

How it works:

  • We fit an "S" Shape logistic Function
  • The curve tells you the porbability if a loan is good or bad
  • If we have a high Credit score, there is a high probability that it's a good loan

SOME OF THE VERY IMP TERMS:

  • Null Deviance - this shows that how the outcome var is predicted by the model given that it only includes the intercept. i.e. there are no explanatory vars in the model</font>.
    • The lower the deviance the better the model</font>
  • Residual Deviance- this shows that how the outcome var is predicted by the model when we HAVE INCLUDED THE PREDICTOR VARS. we should check the deviance of both null and residual deviance and lower should be better

Key Points:¶

  • LL = loglikelihood
  • Null Deviance = 2(LL(Saturated Model) - LL(Null Model)) on df = df_Sat - df_Null
  • Residual Deviance = 2(LL(Saturated Model) - LL(Proposed Model)) df = df_Sat - df_Proposed

The Saturated Model is a model that assumes each data point has its own parameters (which means you have n parameters to estimate.)
The Null Model assumes the exact "opposite", in that is assumes one parameter for all of the data points, which means you only estimate 1 parameter.
The Proposed Model assumes you can explain your data points with p parameters + an intercept term, so you have p+1 parameters.
If your Null Deviance is really small, it means that the Null Model explains the data pretty well. Likewise with your Residual Deviance.
What does really small mean? If your model is "good" then your Deviance is approx Chi^2 with (df_sat - df_model) degrees of freedom

AIC and BIC¶

Akaike information criterion- it is used to measure how good the quality of the model is in comparision to the other models. The lowest val of AIC amongst all models should be selected:

  • AIC = -2(log-likelihood) + 2K

  • K is the number of model parameters (the number of variables in the model plus the intercept).

  • Log-likelihood is a measure of model fit,

  • LIKELIHOOD - tells about how likely is the independent variables impact the outcome variable

Maximum Likelihood¶

Likelihood function: - L of theta. Theta is a vector with 2 values: mean and SD. This is true if we are dealing with a normal/ gaussian distribution.
The L of theta measures the prob of observing a point xi[value in data] if we know the parameters of distribution, in case of normal distribution it will be mean and SD.
Maximum likelihood estimation is a probabilistic framework for automatically finding the probability distribution and parameters that best describe the observed data

Key Terms & Defintions¶

  1. IN a decision tree, every decision rule occurs at a decision node, A branch without a decision rule at the end is called a leaf
  2. Bagging: It is the process of "NOT" taking the entire data set at once, rather creating smaller datasets from the main data set
  3. Similaryl rater than taking all the features, in bagging we take some of the features and continuously change the combinations
  4. ROOT NODE: It represents entire population or sample and this further gets divided into two or more homogeneous sets
  5. Decision Node: When a sub nodes splits into further sub nodes
  6. Pruning: When we remove sub nodes of a decision tree it is called pruning to avoid over fitting
  7. Parent and Child Node: A node which is divided into a sub node is called as parent whereas the sub nodes are the child nodes of parent nodes
    • __RANDOM FOREST__ - __RF divides the entire data into smaller subsets and then further uses them with somethng called as replacement aka Bootstrap Sampling__ - Random Forrest Uses samples of input data with replacement known as "__Bootstrap Sampling__". - Here 1/3rd data is not used for training and can be used for testing. This is known as "__{OOB - Out Of Bag}" Error__. As a result, RF uses Oob hence the need for a separate test data set is removed - HOW TO FIND OPTIMAL NUMBER OF TREES IN THE FOREST: : For regression: Mean Square of error [MSE] : For classification: Gini Index - HOW TO FIND OPTIMAL NUMBER OF PREDCITORS IN THE FOREST: : For regression: one third of obs : For classification : sqrt(k) i.e. square root of number of variables - First try with by default, increase and decrease and whats optimal

Information Gain and Entropy¶

  • The core algorithm for building decision trees called ID3
  • ID3 uses Entropy and Information Gain to construct a decision tree
  • Entropy: ID3 algorithm uses entropy to calculate the homogeneity of a sample [ similar values]
  •  If the sample is completely homogeneous the entropy is zero and if the sample is an equally divided it has entropy of one. As the entropy is a sum probabilities so it goes from 0 to 1
  • Information Gain: IG (Y|X) = H(Y) - H(Y|X)
  • THIS ABOVE formula means, whatis the entropy of Y and what is the entropy of Y given that we know the value of x
  • the bigger the IG the better the model
  • Information gain is used to decide which feature to split on at each step in building the tree
  • The split with the highest information gain will be taken as the first split and the process will continue until all children nodes are pure, or until the information gain is 0.

Impurity Index or Gini Index for DT¶

  1. Gini index says, if we select two items from a population at random then they must be of same class and probability for this is 1
  2. It works with categorical target variable “Success” or “Failure”.
  3. CART uses Gini Method. Higher the Gini Score beter the model
  4. Calculate Gini for sub-nodes, using formula sum of square of probability for success and failure (p^2+q^2).
  5. Calculate Gini for split using weighted Gini score of each node of that split
  6. Per the above formula, below screen examples how to calculate a gini score for split:
    • Calc Gini fr Sub Node = female : Here only 20% females play cricket
    • Calc Gini fr Sub Node = male : Here only 65% females play cricket
    • the formula would be for female: (0.2) (0.2) + (0.8) (0.8) = 0.68
    • the formula would be for male: (0.65) (0.65) + (0.35) (0.35) = 0.55
  7. Now final calc for GINI:
    • (10/30) 0.68 + (20/30) 0.55 = 0.59 # population size = 30, female 10 male =20

Regularization¶

Regularization is a technique which is used to solve the overfitting problem of the machine learning models.
Generally, a good model does not give more weight to a particular feature. The weights are evenly distributed. This can be achieved by doing regularization

/There are two types of regularization as follows:

  • L1 Regularization or Lasso Regularization

  • L2 L2 Regularization or Ridge Regularization </font>
    L1 Regularization or Lasso Regularization adds a penalty to the error function. The penalty is the sum of the absolute values of weights

  • Ridge Regression: along with reduction in error or loss we add 2 more parameters : Lambda * square of slope

  • Lasso Regression: along with reduction in error or loss we add 2 more parameters : Lambda * magnitude of slope

Techniques to Avoid Overfitting¶

  1. Easiest Way is to create train and test environment

  2. Cross Validation [K-fold]: Use your initial training data to generate multiple mini train-test splits. Use these splits to tune your model

  3. Regularization :For example, you could prune a decision tree, use dropout on a neural network, or add a penalty parameter to the cost function in regression

  4. Regularization can be Ridge and Lasso to reduce overfitting

  5. Ensembling: Bagging attempts to reduce the chance overfitting complex models. Boosting attempts to improve the predictive flexibility of simple models
